4.5. Create django app in host server - django 1.5.1

10. Sync db
   python2.7 manage.py syncdb

11. Collect static files
   python2.7 manage.py collectstatic --noinput

12. Install pytz module
   pip install pytz

13. Install sphinx search for django
	First, sphinx search, a later version (0.97 and later)
	http://sphinxsearch.com/downloads.html
	(I'll just provide the .tar)
	
	Next install it.
	http://sphinxsearch.com/docs/current.html#installation
	tar xzvf sphinx-2.0.1-beta.tar.gz
	cd sphinx
	./configure --prefix $HOME # this bit is just if we use webfaction, we
		have no sudo access for the default location (usr/.../.../bin
	make
	make install 

	Install django-sphinx
	
	svn checkout http://django-sphinx.googlecode.com/svn/trunk/
django-sphinx
	Version 2.0 it looks like
	cd django-sphinx
	setup.py install
	Install sphinxapi.py	
	svn checkout http://sphinxsearch.googlecode.com/svn/trunk/
sphinxsearch-read-only
	# Make sure it's installed within the existing sphinx folder!	

	CONFIGURE settings.py

	SPHINX_API_VERSION = 0x113
	SPHINX_SERVER = 'localhost'
	SPHINX_PORT = 3312
	# below is just for django-sphinx :]
	DATABASE_ENGINE = 'mysql'
	DATABASE_HOST = DATABASES['default']['HOST']
	DATABASE_PORT = DATABASES['default']['PORT']
	DATABASE_NAME = DATABASES['default']['NAME']
	DATABASE_USER = DATABASES['default']['USER']
	DATABASE_PASSWORD = DATABASES['default']['PASSWORD']	
	
	ALWAYS be sure to include 'djangosphinx' in the
	Installed apps section of settings.py

	Config the models file as this url states:
	http://justcramer.com/2009/03/25/setting-up-django-and-sphinx-full-text-search-django-sphinx/
	create a directory and a file in the project path with the file in the
directory. Name the dir/file /sphinx_indices/Article_article.spl 

	Run the management command to gen a sphinx.conf file:
	./manage.py generate_sphinx_config cities >> sphinx.conf
	Edit the file so path = /var/data/Article_article matches what you had
above
	also, edit the query so FROM has a \ prior

	Also, add unique searchd configs, like PID file, log files, etc:
	
	searchd
{
    listen = 127.0.0.1:3312
    # log file, searchd run info is logged here
    # optional, default is 'searchd.log'
    log = /home/wintrialucas/webapps/windjango/Wintria/logs/searchd_log.txt

    # query log file, all search queries are logged here
    # optional, default is empty (do not log queries)
    query_log =
/home/wintrialucas/webapps/windjango/Wintria/logs/sphinx_query_log.txt

    pid_file = /home/wintrialucas/webapps/windjango/Wintria/logs/searchd.pid

}

	ALSO IMPORTANT: In the section in the sphinx.conf file, it will ask
for the path to an index:

I used
/home/wintrialucas/webapps/windjango/Wintria/sphinx_indices/Article_article ,
and added the file (empty) Article_article.spl into it. 
(After u run indexer, they get populated, and new files genned).


IMPORTANT: for sphinx, all models must have the pk be an auto incrementing
integer, it must be named 'id'


	Call indexer Article_article --config ~/.../... for initial setup
	
	Run the daemon with: searchd --config ~/.../.../ already has
is_running check

	Great, u indexed it, from now on to just "update" the index instead of
building a new one, call:
	indexer cities --rotate --config ~/.../.../

	THe connection in settings.py should be:
	SPHINX_API_VERSION = 0x113
	SPHINX_SERVER = 'localhost'
	SPHINX_PORT = 3312
